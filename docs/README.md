# Documentation

Main documentation directory for the Khorium MeshGen project.

## Directory Structure

- **adr/**: Architecture Decision Records. Documents why significant design choices were made.
- **templates/**: Standardized templates for project documentation.
  - `DOCUMENTATION_TEMPLATE.md`: Use this for all new features, schemas, and additions.
- **PROJECT_STRUCTURE.md**: Detailed breakdown of the repository organization.
- **AWS_DEPLOYMENT.md**: Guide for deploying the system to AWS infrastructure.

## Documentation Guidelines

1. **Keep it Updated**: When you change code, update the relevant README or create a new feature doc.
2. **Use Templates**: For new features or schema changes, start with `docs/templates/DOCUMENTATION_TEMPLATE.md`.
3. **Be Precise**: Include examples, JSON schemas, and CLI commands where applicable.
